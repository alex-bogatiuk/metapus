# 03. Структура проекта

> **TL;DR:** Проект разделён на 5 основных блоков: `cmd/` (точки входа), `internal/` (закрытый код слоёв), `pkg/` (публичные утилиты), `db/` (миграции) и `frontend/` (Next.js приложение).

> **Тип:** Reference
> **Аудитория:** Developer
> **Связанные:** [02-architecture.md](02-architecture.md), [04-request-lifecycle.md](04-request-lifecycle.md)

---

## 1. Верхний уровень

| Директория / Файл | Назначение |
|-------------------|------------|
| `cmd/` | Точки входа (main). Сборка зависимостей. |
| `configs/` | `.env.example` и файлы конфигурации. |
| `db/` | SQL-миграции базы данных и seed-данные. |
| `internal/` | Основной код приложения. Ограничен для внешнего импорта. |
| `pkg/` | Утилиты, независимые от логики ERP (logger, numerator). |
| `frontend/` | Next.js приложение. |

## 2. Разбор `cmd/`

| Подкаталог | Зачем нужен |
|------------|-------------|
| `server/` | Основной HTTP REST API сервер. |
| `worker/` | Фоновые задачи (outbox, очистка аудита). |
| `tenant/` | CLI-утилита для управления тенантами (создание, миграции). |
| `seed/` | Генератор тестовых данных для разработки. |

## 3. Разбор `db/`

| Подкаталог | Зачем нужен |
|------------|-------------|
| `meta/` | Схема мастер-БД для управления тенантами. |
| `migrations/` | SQL-файлы (goose) для применения к каждой БД тенанта. |
| `seeds/` | Начальные данные (базовые роли, валюты). |

## 4. Разбор `internal/`

| Подкаталог | Слой | Зачем нужен |
|------------|------|-------------|
| `core/` | Core | `BaseEntity`, `AppError`, структуры Multi-tenancy, типы данных. |
| `domain/` | Domain | Бизнес-логика. Папки: `catalogs/`, `documents/`, `registers/`, `reports/`, `posting/`. |
| `content/`| Content | Регистрация встроенных сущностей (Фабрики для реестра расширений). |
| `infrastructure/`| Infra | Реализация БД (`storage/postgres/`), `http/v1/` (роутер, хэндлеры), `cache/`. |

## 5. Фронтенд (`frontend/`)

| Подкаталог | Зачем нужен |
|------------|-------------|
| `app/` | Next.js App Router (страницы, layout). |
| `components/` | Reusable UI компоненты (shadcn/ui + кастомные). |
| `lib/` | Утилиты, клиенты API, `settings-registry`. |
| `hooks/` | React hooks (state, api fetch). |
| `stores/` | Zustand stores (Auth, Tabs). |
