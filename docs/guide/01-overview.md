# 01. Обзор платформы Metapus

> **TL;DR:** Metapus — это современная ERP-платформа для браузера, построенная на Go, PostgreSQL и Next.js. Альтернатива 1С, SAP и Odoo с упором на типизацию и строгую архитектуру.

> **Тип:** Concept
> **Аудитория:** All

## 1. Технологический стек

| Слой | Технологии | Роль |
|------|------------|------|
| **Backend** | Go 1.25, Gin, pgx, squirrel | Бизнес-логика, API, транзакции |
| **Frontend** | Next.js 16, React 19, TypeScript, Tailwind | UI, SSR, маршрутизация |
| **UI Kit** | shadcn/ui, Radix UI, Zustand, React Hook Form | Компоненты, стейт, формы |
| **База данных** | PostgreSQL 16+ | Хранение, изоляция, CDC |

## 2. Манифест архитектуры

Фундаментальные принципы, которым подчиняется весь код в проекте.

### 1. Code is Metadata
Структура данных (Go struct) — единственный источник истины. Платформа не хранит схемы метаданных в JSON-файлах или БД. Код компилируется в контракты (например, `entity.Catalog`), из которых автоматически генерируется UI.

### 2. Explicit over Implicit
Все связи, блокировки транзакций и запросы управляются явно в коде. Система не использует ORM (Object-Relational Mapping). Запросы строятся через `squirrel`, соединения управляются через нативный драйвер `pgx`. 

### 3. Database-per-Tenant
Каждый клиент (тенант) получает изолированную физическую базу данных PostgreSQL. Нет колонки `tenant_id` в таблицах. Это гарантирует абсолютную безопасность данных на уровне БД и упрощает бэкапы.

### 4. Layered Isolation
Используется Clean Architecture. Слой `domain` (бизнес-логика) ничего не знает о слое `infrastructure` (HTTP, база данных) или `presentation`. Зависимости направлены только внутрь.

### 5. Type Safety Everywhere
Отсутствие `interface{}`/`any` в Go там, где можно использовать generics. Отсутствие `any` и `unknown` в TypeScript API-интерфейсах. Контракты (DTO) строго описывают нуллабельность полей.

## 3. Бизнес-модель платформы

Платформа оперирует тремя фундаментальными типами данных (вдохновлено 1С и ERPNext):

1. **Справочники (Catalogs)** — мастер-данные (Номенклатура, Контрагенты, Склады).
2. **Документы (Documents)** — бизнес-события (Поступление товаров, Реализация, Перемещение). Формируют движения.
3. **Регистры (Registers)** — многомерное хранилище (Остатки, Взаиморасчеты, Себестоимость). Отвечают за быстрые аналитические выборки.
