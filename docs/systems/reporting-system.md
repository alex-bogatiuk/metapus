# Система отчётов (Reporting System)

> **TL;DR:** Встроенный Query Engine (аналог СКД из 1С). Разработчик только декларирует Dataset (какие таблицы и поля доступны), а платформа сама строит SQL, генерирует UI (фильтры, колонки), обрабатывает экспорт в XLSX и сохраняет пользовательские варианты отчёта.

> **Тип:** Concept
> **Аудитория:** Developer
> **Связанные:** [filtering.md](filtering.md)

---

## 1. Архитектура Query Engine

Классические отчёты требуют написания сложного SQL для каждого набора фильтров. В Metapus разработчик описывает метаданные отчёта (Dataset), а `Compiler` динамически генерирует SQL на основе того, какие колонки и фильтры выбрал пользователь на фронтенде.

**Процесс выполнения:**
1. **Frontend:** Запрашивает метаданные отчёта (`GET /reports/{key}/metadata`). Платформа возвращает дерево доступных полей (включая связанные справочники).
2. **Frontend:** Пользователь выбирает колонки (например, "Контрагент.ИНН") и настраивает фильтры. Отправляется POST-запрос с `QueryRequest`.
3. **Backend Compiler:**
   - Валидирует запрошенные поля по Whitelist'у.
   - Разрешает ссылки (`Resolve reference paths`). Если запрошено поле "Контрагент.ИНН", автоматически добавляет `LEFT JOIN cat_counterparties`.
   - Применяет `AdvancedFilters` (спускает их в секцию SQL `WHERE`).
   - Генерирует `GROUP BY`, `ORDER BY`, `LIMIT/OFFSET`.
   - Выполняет запрос к PostgreSQL (`pgx`) и отдаёт массив строк.

## 2. Экспорт и Варианты отчётов

Помимо обычного просмотра в виде таблицы:
- **Экспорт (XLSX):** Платформа сама генерирует Excel-файл с учетом группировок (Control Breaks) и форматов данных.
- **Варианты (Variants):** Пользователь может сохранить набор выбранных колонок, сортировок и фильтров как "Вариант отчёта" (аналог пресетов). Варианты хранятся в `sys_report_variants` и могут быть личными или общими для всех пользователей тенанта.

## 3. Как создать новый отчёт

1. Создать хэндлер, реализующий интерфейс платформы.
2. В методе `HandleMeta` вернуть `EntityDef` (список доступных полей).
3. Зарегистрировать хэндлер в `FactoryRegistry` (`reg.RegisterReport(...)`).
4. На фронтенде отчёт появится автоматически по роуту `/reports/{key}`. Никакого React-кода писать не нужно (Metadata-driven UI).
