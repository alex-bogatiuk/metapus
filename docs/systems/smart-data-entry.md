# Каскадное автозаполнение (Smart Data Entry)

> **TL;DR:** В интерфейсе ввода документов (табличных частей) реализован UX-паттерн Smart Advance. Система каскадно заполняет зависимые поля (Ед. изм, НДС) на основе выбранной номенклатуры и автоматически переносит фокус (Tab-skip) на первое пустое поле, требующее ручного ввода (Кол-во).

> **Тип:** Concept
> **Аудитория:** Frontend Developer

---

## 1. Философия Smart Advance

При потоковом вводе документов пользователь не должен нажимать `Tab` впустую. Если система может вывести значение сама — она заполняет поле и перепрыгивает его.

**Правило фокуса:** После выбора значения фокус перемещается на первое поле в строке, которое **невозможно определить автоматически** из уже заполненных данных.

**Пример (Поступление товаров):**
1. Выбрана `Номенклатура`.
2. Система подтягивает `baseUnitId` и заполняет "Ед. изм".
3. Подтягивает `defaultVatRateId` и заполняет "Ставка НДС".
4. Система сканирует DOM строки (начиная слева направо), видит, что "Кол-во" пустое, и автоматически делает `focus()` на него.

Если у выбранного товара *нет* базовой единицы измерения в карточке, каскад не срабатывает, и фокус упадет на поле "Ед. изм", заставляя пользователя выбрать ее вручную.

## 2. Архитектура Frontend-реализации

1. **Каскад:** Логика вызова API номенклатуры и заполнения формы инкапсулирована в `handleProductSelect` (внутри хуков типа `useDocumentLineActions`). Возвращает `Promise`.
2. **Сканирование DOM:** Утилита `advanceToFirstEmptyField(row)` из `lib/table-field-navigation.ts` после завершения каскада обходит все `<td>` строки. Для Inputs проверяется `value`, для Combobox проверяется атрибут `data-has-value`.
3. **Prop `autoAdvance`:** Если у `ReferenceField` установлен `autoAdvance={true}`, то после ручного выбора элемента (клик мышкой) фокус автоматически "провалится" в следующее поле вправо. Этот проп **НЕ** ставится на поле Номенклатуры (потому что там сложный асинхронный каскад), но ставится на поля вроде "Ставка НДС".
