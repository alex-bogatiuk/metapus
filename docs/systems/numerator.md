# Автонумерация (Numerator)

> **TL;DR:** Сервис автогенерации кодов для справочников и номеров для документов. Поддерживает настройку префиксов, включение года и два режима производительности (Strict и Cached).

> **Тип:** Concept
> **Аудитория:** Developer
> **Связанные:** [crud-pipeline.md](crud-pipeline.md)

---

## 1. Конфигурация нумератора

Формат номеров задаётся объектом `Config`:
- `Prefix`: "INV", "GR", "CP" (Счет, Поступление, Контрагент).
- `IncludeYear`: `true` добавит год, например `INV-2024-00001`.
- `PadWidth`: ширина числовой части с ведущими нулями (5 -> `00001`).
- `ResetPeriod`: как часто сбрасывать счётчик ("year", "month", "never").

## 2. Стратегии выдачи (Strategies)

### Strict Strategy (Строгая)
Гарантирует **отсутствие пропусков (gaps)** в номерах.
- Делает SQL `UPDATE sys_sequences` для **каждого** номера.
- Оптимально для строгой бухгалтерской отчётности (например, Счета-фактуры).

### Cached Strategy (Кешированная)
Гарантирует **максимальную производительность**.
- Делает SQL запрос для резервирования блока (например, 50 номеров) за раз, а затем раздаёт их из памяти (Lock-free fast path).
- **Минус:** при перезапуске сервера неиспользованные номера из зарезервированного блока "сгорают" (образуются пропуски).
- Оптимально для массового импорта или внутренних технических документов.

## 3. Интеграция через Хуки

Автонумератор встроен в стандартный CRUD pipeline:
1. Вызывается в хуке `BeforeCreate`.
2. **Проверка заполненности:** Если пользователь передал `Code` (справочник) или `Number` (документ) вручную в JSON — нумератор пропускается.
3. **Вне бизнес-транзакции:** Нумератор делает запросы в БД через Pool напрямую (вне контекста `RunInTransaction`). Это нужно, чтобы избегать блокировки таблицы `sys_sequences` на время выполнения долгой бизнес-транзакции.

## 4. Tenant-Aware (Мультитенантность)

Поскольку `Cached Strategy` хранит пулы номеров в оперативной памяти (RAM) сервера, ключи кеширования включают `TenantID` (например: `5cfe45cb...:INV_2024`). Это предотвращает случайную выдачу номера тенанта А документу тенанта Б.
