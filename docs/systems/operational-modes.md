# Операционные режимы (Operational Modes)

> **TL;DR:** Metapus всегда использует физическую изоляцию `Database-per-Tenant`. Однако кластер серверов может быть развернут в трех разных режимах: Cloud SaaS (несколько версий одновременно), Self-Hosted (мультитенант на одной версии) и Dedicated (монопольно).

> **Тип:** Concept
> **Аудитория:** DevOps, Architect
> **Связанные:** [multi-tenancy.md](multi-tenancy.md), [cloud-deployment.md](../howto/cloud-deployment.md)

---

## 1. Общие принципы

В бизнес-коде (Domain/HTTP/Repo) **никогда** не пишутся SQL-запросы с `WHERE tenant_id = ?`. Изоляция происходит на уровне инфраструктуры (Middleware выбирает пул соединений к конкретной базе и кладет его в `context.Context`).

Shared DB подход (данные разных клиентов в одной таблице) **не поддерживается**, так как несовместим с нашей моделью расширений (Custom Fields) и высокими требованиями к безопасности.

## 2. Cloud SaaS (Multi-Version Mode)

Облачный режим для публичного SaaS.
- **Версионирование:** Можно держать в кластере одновременно сервера версий `v1.3` и `v1.4`.
- **Маршрутизация:** Nginx проверяет мета-базу `tenants` (какая версия закреплена за клиентом) и отправляет запрос на нужный пул серверов.
- **Активация:** Через переменную `VERSION_GROUP=v1.3.0` у бинарника. Запрос от тенанта другой версии будет отклонен с `421 Misdirected Request`.

## 3. Self-Hosted (Standalone Multi-Tenant)

Режим для крупных холдингов с множеством юрлиц (тенантов).
- Все базы (дочерние компании) крутятся на **одной** версии сервера. Nginx просто балансирует трафик (Round Robin).
- Обновление системы происходит синхронно: `go run cmd/tenant/main.go migrate` прогоняет миграции для всех БД по очереди, затем кластер рестартуется.
- **Активация:** Переменная `VERSION_GROUP` не устанавливается.

## 4. Dedicated (Single Tenant)

Система работает как классическая монопольная ERP для одной компании.
- В мета-базе создается ровно 1 запись тенанта.
- Фронтенд или прокси жестко прикрепляет `X-Tenant-ID` ко всем запросам.
- Пользователь вообще не видит мультитенантных механизмов в UI.
- Идеально для обновления через [Updater Agent](updater-agent.md).
