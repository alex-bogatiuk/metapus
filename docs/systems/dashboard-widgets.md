# Дашборды и Виджеты (Dashboard Widgets)

> **TL;DR:** Система виджетов реализована полностью на фронтенде. Состояние раскладки (Grid Layout) сохраняется в JSONB-колонку `user_preferences`.

> **Тип:** Concept
> **Аудитория:** Frontend Developer

---

## 1. Архитектура

Бэкенд Metapus ничего не знает о виджетах. Дашборд — это чисто клиентская фича.
Состояние (расположение карточек, размеры, настройки) сохраняется в БД в таблице `user_preferences` (колонка `dashboard_layout`).

Каждый виджет состоит из 3 частей:
1. **Zod Схема (`types/dashboard.ts`):** Описывает настройки виджета (например, `limit` или `warehouseId`).
2. **Запись в Реестре (`lib/widget-registry.ts`):** Метаданные (иконка, название, категория, доступные размеры).
3. **Рендерер (React Component):** Компонент, который запрашивает данные с бэкенда через хук `useWidgetData` и рендерит их.

## 2. Размеры виджетов

Дашборд использует 4-колоночную сетку. Размер задаётся строкой `{ширина}x{высота}`:
- `2x1` — Мини KPI (занимает половину ширины).
- `4x2` — Широкий список (на всю ширину).
- `4x4` — Большой график.

При регистрации виджета (в `allowedSizes`) указывается список разрешённых размеров.

## 3. Жизненный цикл и Права доступа

**Авто-обновление:**
Рендерер использует хук `useWidgetData`, который автоматически:
- Делает поллинг (перезапрос) данных с заданным интервалом (например, раз в минуту).
- Ставит опросы на паузу, когда дашборд находится в режиме "Редактирования" (Edit Mode).

**Безопасность (Права):**
В реестре виджета указывается `requiredPermission` (например, `"report:stock:read"`). Если у пользователя нет этого права, виджет просто **не будет отображаться в Галерее виджетов**. Однако физическая защита данных всё равно происходит на бэкенде — если кто-то попытается "хакнуть" UI и добавить виджет, бэкенд вернёт 403 Forbidden.
