# Система печатных форм (Printing System)

> **TL;DR:** Code is Metadata. Шаблоны печатных форм (`.gohtml`) хранятся в репозитории как код. Бэкенд генерирует HTML, который фронтенд может рендерить, либо конвертирует его в PDF/DOCX.

> **Тип:** Concept
> **Аудитория:** Developer

---

## 1. Архитектура: Code is Metadata

Печатные формы не хранятся в базе данных. Они описываются в статическом реестре (`internal/domain/printing/registry.go`). Это дает строгую типизацию, версионирование через Git и защищает от поломок при миграциях БД.

### Определение формы
Каждая форма (`PrintFormDef`) привязывается к типу документа и имеет:
- Идентификатор (например, `"proforma_invoice"`)
- Имя файла шаблона (`template.gohtml`)
- Размер бумаги (`A4`, `A5`)
- Категорию (`standard` — системная, `custom` — добавленная интегратором).

## 2. Рендеринг и Шаблоны

Шаблоны пишутся на стандартном Go `html/template`. Данные документа передаются в шаблон через структуру (DTO).

Платформа поддерживает несколько форматов вывода:
1. **HTML:** Возвращается чистый HTML. Фронтенд открывает его в новом окне браузера и вызывает `window.print()`.
2. **PDF:** Бэкенд использует Headless Browser (Chrome/Edge через CDP) для конвертации отрендеренного HTML в PDF с сохранением форматирования (`@media print`).
3. **DOCX:** Генерация документов Word (на базе XML-шаблонов внутри `.docx` архива).

## 3. Frontend Интеграция

На фронтенде используется универсальный компонент `PrintMenuButton`.
Он запрашивает у бэкенда (`GET /api/v1/document/{type}/print-forms`) список доступных форм для конкретного документа.

Кнопка автоматически:
- Группирует формы по категориям (Регламентированные / Кастомные).
- Рендерит иконки для мгновенного скачивания в нужных форматах (🖨 HTML, 📄 PDF, 📃 DOCX).
