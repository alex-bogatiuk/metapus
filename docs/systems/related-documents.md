# Связанные документы (Related Documents)

> **TL;DR:** Система автоматически строит дерево подчиненности документов (на основе `basis_id`). Карточки предпросмотра (Preview Cards) также генерируются автоматически, извлекая ссылочные поля через Metadata Inspector.

> **Тип:** Concept
> **Аудитория:** Developer

---

## 1. Карточки предпросмотра (Preview Cards)

В интерфейсе "Связанные документы", при наведении на документ всплывает карточка с ключевой информацией (Контрагент, Склад и т.д.). Разработчику **не нужно настраивать эти карточки вручную**.

**Zero-Config механизм:**
1. При запуске сервера `metadata.Inspector` сканирует структуру каждого документа.
2. Любое поле типа `id.ID`, ссылающееся на справочник, автоматически помечается как поле для превью (например, `SupplierID`). Имя (Label) берется из тега `meta:"label:Поставщик"`.
3. Системные ссылки (`created_by`, `basis_id`) исключаются.
4. Чтобы скрыть бизнес-ссылку из превью вручную, используйте тег `meta:"preview:false"`.

В репозитории генерируется один большой SQL с динамическими `LEFT JOIN` для всех превью-полей, избегая проблемы N+1.

## 2. Дерево подчиненности (Hierarchy Tree)

Дерево строится на основе полей `basis_type` и `basis_id` (Документ-основание).

**Алгоритм обхода:**
1. Система сначала рекурсивно идет **вверх** по ссылкам `basis_id`, чтобы найти "Корень" всей цепочки документов.
2. От найденного корня строится полное дерево **вниз** (поиск всех документов, ссылающихся на текущий узел).
3. Применяются лимиты безопасности (например, максимальная глубина дерева 10), чтобы избежать зацикливаний и чрезмерной нагрузки на БД.
