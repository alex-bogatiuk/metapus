# Обзор платформы Metapus

> **Metapus** — современная платформа для построения бизнес-приложений на Go, альтернатива классическим ERP-системам (1С:Предприятие, Odoo, ERPNext), работающая исключительно в веб-браузере.

---

## Ключевые характеристики

| Характеристика | Значение |
|----------------|----------|
| **Backend** | Go (Golang) + Gin Framework |
| **Frontend** | Next.js + React + TypeScript + shadcn/ui |
| **База данных** | PostgreSQL 16+ с отдельными таблицами для каждого типа |
| **API** | RESTful с версионированием (`/api/v1`, `/api/v2`) |
| **Архитектура** | Clean Architecture + Vertical Slices |
| **Multi-tenancy** | Database-per-Tenant (отдельная БД для каждого клиента) |
| **Аутентификация** | JWT (Access + Refresh tokens) |

---

## Манифест платформы

Семь фундаментальных принципов, определяющих все архитектурные решения:

### 1. CODE IS METADATA
Структура данных (Go struct) — **единственный источник истины**. Метаданные системы выражаются компилируемыми Go-структурами и контрактами, а не JSON-описаниями схем в БД.

### 2. EXPLICIT OVER IMPLICIT
Все связи, блокировки и транзакции управляются **явно** в коде. Никакой "магии" ORM.

### 3. PERFORMANCE FIRST
Нативные драйверы (pgx), пулы соединений, MinorUnits/Quantity для точности вычислений.

### 4. LAYERED ISOLATION
Domain ничего не знает о Infrastructure и Presentation. Зависимости направлены **только внутрь**.

### 5. IMMUTABLE LEDGER
Движения регистров **никогда не обновляются** (`UPDATE`). При изменении документа создаётся новая версия движений.

### 6. DATABASE-PER-TENANT
Multi-tenancy через отдельные PostgreSQL базы данных. Физическая изоляция без фильтрации по tenant в запросах.

### 7. NAMING IS CONTRACT
Все имена подчиняются единому стандарту. Нарушение = отклонение. Подробнее: [15-naming-conventions.md](15-naming-conventions.md).

## Приоритеты при проектировании

1. **Расширяемость** — поведение задаётся конфигурацией/метаданными и расширяемыми точками (хуки, интерфейсы), а не хардкодом
2. **Читаемость** — Clean Architecture + уместный DDD
3. **Производительность** — простые и эффективные решения
