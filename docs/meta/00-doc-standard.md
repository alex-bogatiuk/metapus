# 00. Стандарт документации Metapus

> **TL;DR:** Документация группируется по намерению читателя в 4 типа (Concept, How-To, Reference, Rule). Стиль — UX-редактор (коротко, по делу, для сканирования). Оптимизировано для ИИ-агентов.

> **Тип:** Rule
> **Аудитория:** Developer, AI-Agent
> **Связанные:** [ROUTER.md](../ROUTER.md)

---

## 1. Таксономия документов

Документы группируются по намерению читателя, а не по архитектурному слою (фронтенд/бэкенд). Каждый тип лежит в своей директории.

| Директория | Тип документа | Вопрос читателя | Стиль | Пример |
|------------|---------------|-----------------|-------|--------|
| `guide/` | **Concept** (Объяснение) | «Что это и зачем?» | Повествовательный | Архитектура, Multi-tenancy |
| `systems/` | **Concept** (Глубокое) | «Как работает подсистема X?» | Технический, диаграммы | Posting Engine, Reporting |
| `howto/` | **How-To** (Рецепт) | «Как мне сделать X?» | Шаги, копипастабельный код | Добавить справочник |
| `reference/`| **Reference / Rule** | «Какой формат? Что запрещено?» | Таблицы, списки, ✅/❌ | Именование, API, правила |

> [!IMPORTANT]
> Один документ = один тип. Если документ содержит и теорию «зачем», и практику «как» — разделите его на два связанных файла.

## 2. Анатомия документа (Шаблон)

```markdown
# Название (Subtitle)

> **TL;DR:** Одно предложение — факт или суть.
> **Тип:** Concept | How-To | Reference | Rule
> **Аудитория:** Developer | AI-Agent | Admin | All
> **Связанные:** [link.md](link.md)

---

## 1. Первая секция
Факт. Таблица. Код.

## N. Файловая карта
\`\`\`path
internal/domain/file.go — описание
\`\`\`

## N+1. Чеклист (только для How-To)
- [ ] Сделал А
- [ ] Сделал Б

## Связанные документы
- [file.md](file.md) — одно предложение зачем читать.
```

## 3. Стиль UX-редактора

Документацию сканируют. Понижайте когнитивную нагрузку.

| Правило | ✅ Правильно | ❌ Неправильно |
|---------|-------------|---------------|
| Заголовок | Существительное (Курсорная пагинация) | Процесс (Как работает пагинация) |
| Вступление | Факт (Курсор — это JSON) | Вода (В этом разделе мы рассмотрим...) |
| Форматирование | Таблицы, код | Длинные списки и абзацы текста |
| Модальность | Факт (Система делает) | Предположение (Система должна делать) |
| Вежливость | Нулевая | Пожалуйста, рекомендуется |

**Запрещённые фразы:**
- «Важно отметить, что...» (используйте `> [!IMPORTANT]`)
- «Как было сказано выше...» (используйте якорные ссылки)
- «И так далее» (используйте явные перечисления)

---

## 4. Оптимизация для ИИ-агентов

Документация должна быть машиночитаемой для ИИ-агента.

1. **`ROUTER.md`** — единственный входной файл. Агент ищет по таблице ключей.
2. **Файловые карты** — явные пути к файлам в кодовых блоках (агенту не нужно искать).
3. **Чеклисты** — маркеры завершения работы для агента.
4. **Compile-ready код** — без `...` и псевдокода, если это пример для копирования. Первая строка кода всегда путь: `// internal/domain/model.go`.
