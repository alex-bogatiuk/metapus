# Материалы дипломного проекта Metapus

В данном каталоге содержатся отчетные материалы по учебной практике, спроектированные на базе многопользовательской ERP-системы **Metapus**.

Все тексты прошли UX-редактуру, избавлены от рекламных формулировок и оформлены в строгом академическом стиле.

---

## 📂 Структура каталога

Каталог организован следующим образом:

1. **[Задание 1. Описание тематики дипломного проекта](./diploma_task1_description.md)**
   * Теоретический отчет, содержащий обоснование актуальности темы, определение целевой аудитории, описание функционала, стека технологий (*Go 1.25, Next.js 16, PostgreSQL 17*) и ожидаемых результатов.
   * *Файл:* `diploma_task1_description.md`

2. **[Задание 2. Проектирование use-case диаграммы](./diploma_task2_usecase.md)**
   * Спецификация прецедентов использования системы для 4 ключевых ролей (*Superadmin, Tenant Admin, Operator, Client*).
   * Содержит интерактивный Mermaid-код диаграммы связей и графический UML-макет.
   * *Файл:* `diploma_task2_usecase.md`
   * *Графическая диаграмма:* `use_case_diagram.png`

3. **[Задание 3. Проектирование и разработка интерфейса](./diploma_task3_interface.md)**
   * Описание структуры веб-интерфейса Next.js 16, применения HTML5 семантики и адаптивности Tailwind CSS.
   * Содержит примеры кода валидации форм с помощью Zod и React Hook Form, а также логику отслеживания dirty-state вкладок через Zustand.
   * Включает пошаговое руководство по локальному развертыванию всей системы.
   * *Файл:* `diploma_task3_interface.md`
   * *Визуальные макеты экранов:*
     * `dashboard_mockup.png` — Панель мониторинга (Dashboard).
     * `catalog_list_mockup.png` — Страница справочника с фильтрацией.
     * `document_form_mockup.png` — Форма создания и проведения складского документа.

## 💻 Проверка качества кода

Кодовая база системы полностью валидна. Для проверки корректности типов, компиляции и соблюдения стандартов кодирования вы можете выполнить следующие команды:

```bash
# Проверка и тесты серверной части (Go 1.25)
go build ./...
go test ./...

# Тайпчекинг и линтинг клиентской части (Next.js 16 / React 19)
cd ../frontend
npx tsc --noEmit
npm run lint
```
